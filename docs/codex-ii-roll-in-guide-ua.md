# Codex II — посібник із впровадження

Аудиторія: майстри гри, гравці та адміністратори світів, які переводять світ Star Wars FFG у Foundry VTT на аркуші Codex II.

Це практичний посібник із впровадження. Повний перелік можливостей дивіться у файлі [`codex-ii-theme-features.md`](codex-ii-theme-features.md).

## 1. Що таке Codex II

Codex II — це додаткова сім'я аркушів для системи Star Wars FFG. Вона змінює вигляд і зручність роботи з аркушами акторів та предметів, але залишає ті самі дані персонажів, предмети, пули кубів, досвід, модифікатори, активні ефекти та ігрову логіку.

Варто використовувати Codex II, якщо ви хочете:

- компактніший і спеціально спроєктований аркуш персонажа;
- кольорові схеми окремо для акторів або предметів;
- зручніші картки інвентарю;
- окремі макети для міньйонів і транспортних засобів;
- дерева розвитку Codex для силових здібностей, спеціалізацій і фірмових здібностей.

Codex II не потребує перестворення персонажів. Наявні актори та предмети можна відкривати через аркуші Codex, якщо їхній тип підтримується.

## 2. Перед впровадженням

Рекомендовані перевірки перед перемиканням живої групи:

1. Переконайтеся, що світ працює на Foundry VTT v13 або новішій версії.
2. Переконайтеся, що збірка системи Star Wars FFG використовує сучасний шлях аркушів ApplicationV2 / DocumentSheetV2. Codex II не працює на legacy-збірках, де доступні лише V1-аркуші.
3. Переконайтеся, що збірка системи Star Wars FFG містить Codex II.
4. Зробіть резервну копію світу перед зміною стандартних аркушів.
5. Виберіть одного тестового актора з повним інвентарем, талантами, критичними травмами та, за можливості, силовими здібностями.
6. Спочатку відкрийте цього актора через Codex II і перевірте, що дані виглядають правильно.
7. Поясніть гравцям, що модель даних та сама; змінюється лише інтерфейс аркуша.

## 3. Глобальне ввімкнення Codex II

Щоб нові або не налаштовані підтримувані документи відкривалися як Codex II:

1. Відкрийте Game Settings.
2. Відкрийте Configure Settings.
3. У розділі Star Wars FFG натисніть кнопку **Configure Codex**. Вона відкриває окреме вікно Codex Settings — Default Sheet Theme більше не показане прямо у звичайному списку налаштувань.
4. Знайдіть налаштування Default Sheet Theme.
5. Виберіть один із варіантів Codex II:
   - Codex II - Republic
   - Codex II - Empire
   - Codex II - Dark
   - Codex II - Light
   - Codex II - Mercenary
   - Codex II - Eldritch Horror - Scholar
   - Codex II - Eldritch Horror - Fate
6. Перезавантажте Foundry, якщо вона цього попросить або якщо аркуші не змінилися одразу.

Важливо: це клієнтське налаштування. Кожен користувач може мати власне значення Default Sheet Theme. Саме вікно Configure Codex доступне всім користувачам, а не лише майстру гри, — бо вибір теми особистий. Але в тому самому вікні є перемикач **Advantages Heal Strain (House Rule)**, який бачить і змінює лише майстер гри, бо він впливає на весь стіл (див. розділ 9, «Відновити strain»).

## 4. Увімкнення Codex II для окремого актора або предмета

Для конкретного актора або предмета:

1. Відкрийте аркуш актора або предмета.
2. Відкрийте конфігурацію аркуша для цього документа.
3. Виберіть Codex II Sheet для підтримуваних акторів або Codex II Item Sheet для підтримуваних предметів.
4. Збережіть конфігурацію аркуша.

Вибір аркуша для конкретного документа має пріоритет над Default Sheet Theme. Це корисно, якщо один гравець хоче Codex II, а інший актор має залишитися на стандартному аркуші.

## 5. Підтримувані типи акторів

Codex II підтримує такі аркуші акторів:

- Character
- Rival
- Nemesis
- Minion
- Vehicle
- Character-type Adversary Sheet

Актори Homestead не входять до набору акторських аркушів Codex II.

## 6. Підтримувані типи предметів

Codex II підтримує такі аркуші предметів:

- Weapon
- Armour
- Gear
- Talent
- Force power
- Specialization
- Signature ability
- Ship weapon
- Item attachment
- Ship attachment
- Ability
- Critical injury
- Critical damage
- Obligation
- Motivation
- Background
- Homestead upgrade

Ці типи залишаються на стандартних аркушах:

- Career
- Species
- Item modifier

## 7. Вибір кольорових схем

Default Sheet Theme задає стандартну кольорову схему Codex для документів без власного налаштування.

Щоб змінити схему для окремого актора або предмета:

1. Відкрийте аркуш актора або предмета Codex.
2. Скористайтеся кнопкою палітри у заголовку вікна.
3. У вікні виберіть схему з групи **Modern** (Republic/Empire/Dark/Light/Mercenary) або з групи **Eldritch Horror** (Scholar/Fate).
4. Вибір буде збережено на цьому акторі або предметі одразу.

Доступні схеми:

- Republic
- Empire
- Dark
- Light
- Mercenary
- Eldritch Horror – Scholar
- Eldritch Horror – Fate

Обидва варіанти Eldritch Horror отримують у заголовку унікальний процедурний сигіл (візерунок), згенерований окремо для кожного актора чи предмета: Scholar показує один медальйон-сигіл на паперовій текстурі, Fate — щільнішу стіну сигілів на мармуровій текстурі. Актори чи предмети зі старою, ще не розділеною схемою `eldritch` автоматично відкриваються як Scholar.

Предмети успадковують схему власного актора, якщо сам предмет не має окремої схеми.

## 8. Короткий тур для гравців

Під час знайомства гравців із Codex II спочатку покажіть ці області:

- Header: портрет, ім'я, вид, кар'єра, пігулки спеціалізацій/Сили/фірмових здібностей, XP і кредити.
- Skills: характеристики, чип Сили, якщо ввімкнено, рядки навичок, пули кубів і ранги.
- Combat: зброя та броня.
- Gear: інвентар, спорядження та кількість.
- Talents: таланти, здібності, силові здібності та фірмові здібності.
- Bio: біографія, описові поля, обов'язок/мораль/конфлікт.
- XP: суми XP і журнал, лише для персонажів.
- AEs: активні ефекти.

Для міньйонів насамперед покажіть Group Strength і Combined Wound Pool.

Для транспорту покажіть швидкість, пошкодження корпусу, системне напруження, чотиризонні щити, екіпаж, зброю, модулі та вантаж.

## 9. Типові дії гравців

### Кинути навичку

Використовуйте пул кубів у рядку навички. Картки зброї також показують відповідну навичку та пул; натисніть іконку або зображення зброї, щоб кинути атаку.

### Купити за XP

На аркушах персонажів:

1. Натисніть чип XP у заголовку.
2. З'являться іконки купівлі там, де доступні покупки.
3. Натисніть іконку купівлі для навички, характеристики, спеціалізації, таланту, силової здібності або фірмової здібності.
4. Натисніть чип XP ще раз, щоб приховати кнопки купівлі.

Режим купівлі за XP тимчасовий і скидається після закриття аркуша. Він вимикається, коли активний режим редагування.

### Змінити кредити

Використовуйте чип CR у заголовку.

- Натисніть поле кредитів, щоб ввести повне значення вручну.
- Натисніть Change, щоб відкрити панель додавання/віднімання.
- Виберіть плюс або мінус, введіть суму й підтвердьте.

Відображення використовує розділювачі тисяч, але збережене значення лишається числовим.

### Змінити рани або strain

Використовуйте кнопки плюс/мінус у панелях Wounds або Strain. Якщо маєте дозвіл, можна також редагувати поточне значення та поріг вручну.

Рани й strain можуть перевищувати поріг. Codex лише не дозволяє опустити їх нижче нуля.

### Відновити strain

Натисніть кнопку відновлення strain у панелі Strain.

Codex відкриє власне вікно вибору пулу кубів Cool або Discipline, попередньо вибираючи статистично сильніший пул. Киньте його — і Codex зніме strain, що дорівнює кількості success. Якщо майстер гри увімкнув хаусрул **Advantages Heal Strain** (типово увімкнено), кожні 2 advantage додатково знімають ще 1 strain. Результат публікується в чат, а значення strain актора оновлюється автоматично.

Ця функція більше не залежить від модуля token-action-hud-ffgsw — вікно відновлення тепер повністю вбудоване в Codex. Якщо в актора немає strain для відновлення, або йому бракує навички чи характеристики Cool/Discipline, замість вікна з'явиться сповіщення.

### Екіпірувати або нести предмети

Картки зброї та броні мають перемикач екіпірування.

Картки зброї, броні та спорядження також мають перемикач carried. Вимикайте carried, коли предмет залишено на кораблі, базі, у схованці або деінде й він не має рахуватися до encumbrance.

### Перевпорядкувати картки інвентарю

Якщо ви можете редагувати актора, перетягніть маленьку ручку на картці зброї, броні або спорядження. Картки можна перевпорядковувати тільки всередині своєї категорії.

### Використовувати боєприпаси

Якщо для зброї ввімкнено ammo, її розгорнута картка може показувати чип ammo. Використовуйте плюс/мінус, щоб змінити поточні боєприпаси без згортання картки.

### Відкрити дерева

Натискайте пігулки або картки спеціалізацій, силових здібностей чи фірмових здібностей, щоб відкрити їхні аркуші Codex. Стоси з кількох пігулок спочатку розгортаються; потім вибір пігулки відкриває відповідне дерево.

## 10. Чеклист майстра гри

Для плавного впровадження в кампанії:

1. Зробіть резервну копію світу.
2. Встановіть власний Default Sheet Theme на одну зі схем Codex II (Configure Settings → Star Wars FFG → Configure Codex).
3. Відкрийте тестового персонажа, міньйона та транспорт.
4. Перевірте картки предметів, кидки, покупки за XP та активні ефекти.
5. Вирішіть, чи підходить вашому столу хаусрул **Advantages Heal Strain** (те саме вікно Configure Codex, типово увімкнено).
6. Оберіть стандартну схему кампанії.
7. Вирішіть, чи можуть гравці самі вибирати схеми акторів.
8. Для кожного персонажа відкрийте аркуш і, за потреби, задайте схему актора.
9. Переконайтеся, що користувачі Сили мають увімкнений Force Pool, якщо мають силові здібності.
10. Перевірте encumbrance після використання перемикача carried.
11. Навчіть гравців користуватися чипом XP, перемикачем carried і стосами пігулок.

## 11. Рекомендований план впровадження

### Фаза 1: проба майстра гри

Використовуйте Codex II приватно як майстер гри протягом однієї сесії підготовки або пробної гри. Перевірте NPC, транспорт і копію одного персонажа гравця.

### Фаза 2: попередній перегляд для гравців

Попросіть гравців відкрити своїх персонажів у Codex II. Нехай виберуть схему й переглянуть вкладки. Не просіть їх одразу робити незворотні зміни.

### Фаза 3: жива сесія

Використовуйте Codex II за столом для звичайних кидків, ран, strain, спорядження та XP. Тримайте стандартні аркуші як запасний варіант.

### Фаза 4: прибирання

Після сесії виправте акторів із неправильним вибором аркуша, відсутнім Force Pool або предметами, які треба позначити як not carried.

## 12. Дерева розвитку

Codex II має деревоподібні аркуші для force powers, specializations і signature abilities.

Для гравців:

- Вивчені вузли підсвічуються.
- Іконки купівлі з'являються лише тоді, коли вузол можна купити.
- Лінії світяться, коли обидва з'єднані вузли вивчені.
- Описи можна відкривати або редагувати через popout editors, якщо є дозвіл.

Для майстрів гри або авторів:

- Натисніть кнопку пера, щоб увійти в режим редагування дерева.
- Режим редагування тимчасовий і діє лише для поточного вікна.
- Вузли force power і signature ability можна об'єднувати або розділяти, коли це дозволено.
- Перемикачі верхніх і правих з'єднань з'являються в режимі редагування.
- Signature abilities також мають uplink bar над базовою здібністю.
- Specialization trees мають вкладку Skills для career skills і перемикача Universal.

## 13. Міньйони

Аркуш міньйона використовується інакше, ніж аркуш персонажа:

- Group Strength показує живих учасників і максимальний розмір групи.
- Кнопка мінус прибирає одного учасника, зберігаючи часткові рани поточного учасника.
- Кнопка плюс повертає одного учасника, якщо це можливо.
- Wipe Out ліквідує всю групу.
- Combined Wound Pool показує сегменти ран, згруповані за учасниками.
- Wounds per member можна редагувати в режимі редагування.
- Group skills використовують перемикачі group-skill замість career-skill.

## 14. Транспорт

Аркуш транспорту організований навколо транспортної гри:

- Speed, silhouette, handling і armour показані верхніми чипами.
- Hull Trauma і System Strain використовують ті самі степери й треки, що й Wounds/Strain у персонажів.
- Four-zone defense показує щити fore, aft, port і starboard навколо зображення транспорту.
- Systems panel тримає разом hyperdrive, sensors, consumables, navicomputer, crew count, passenger capacity, hardpoints і space-vehicle status.
- Weapon Systems показує ship weapons.
- Attachments показує ship attachments і використання hardpoints.
- Crew показує ролі екіпажу й елементи кидків.
- Cargo містить некорабельний інвентар.

## 15. Усунення проблем

### Гравець не бачить Codex II

Перевірте, чи його клієнтський Default Sheet Theme досі не стоїть на Default. Це налаштування окреме для кожного клієнта. Також перевірте, чи актор не має явно вибраного не-Codex аркуша.

### Не можу знайти Default Sheet Theme в Configure Settings

Це налаштування переїхало у власне вікно. Відкрийте Configure Settings → Star Wars FFG → **Configure Codex** — Default Sheet Theme буде всередині цього вікна.

### Актор відкривається з неправильною схемою Codex

Відкрийте кнопку палітри у заголовку аркуша й виберіть схему. Схеми, збережені на акторі, мають пріоритет над стандартним налаштуванням.

### Предмет відкривається з іншою схемою, ніж актор

Предмет може мати власну scheme flag. Відкрийте палітру предмета й виберіть потрібну схему або вручну вирівняйте її зі схемою актора.

### Force powers є, але force pool відсутній

Увімкніть force pool в опціях або конфігурації актора. Codex показує попередження на вкладці Talents, коли силові здібності існують, але Force Dice Pool вимкнено.

### Encumbrance здається завеликим

Перевірте перемикачі carried на зброї, броні та спорядженні. Предмети, позначені як not carried, не рахуються до encumbrance.

### Активні ефекти спорядження здаються вимкненими

Перемкніть carried у предмета gear вимкнено й знову ввімкнено. Стан carried синхронізує активні ефекти цього предмета.

### Кнопка відновлення strain показує помилку замість вікна вибору

Актору бракує навички чи характеристики Cool або Discipline — вбудоване відновлення strain у Codex потребує обох. Додайте акторові відсутню навичку/характеристику.

### Codex sheet не з'являється в конфігурації аркуша

Переконайтеся, що тип документа підтримується. Career, species, item modifier і homestead actor sheets не зареєстровані для Codex.

### Елементи редагування дерева відсутні

Натисніть кнопку пера на аркуші дерева. Режим редагування тимчасовий і не є тим самим, що стандартний збережений edit state.

### Іконка купівлі відсутня

Перевірте, чи активний XP-buy mode на акторському аркуші, або чи базовий предмет дерева належить актору й вузол можна купити. Також перевірте edit mode: на акторських аркушах XP-buy mode вимикається, коли активний edit mode.

## 16. Коротка пам'ятка для гравців

Можна дати гравцям цю коротку версію:

1. Використовуйте вкладки посередині аркуша для навігації.
2. Натискайте XP box, щоб показати або сховати кнопки купівлі.
3. Натискайте зображення зброї, щоб кидати атаки.
4. Використовуйте іконки валізи, щоб позначати спорядження як carried або left behind.
5. Використовуйте плюс/мінус для ран, strain, ammo та подібних лічильників.
6. Натискайте stacked pills для спеціалізацій, силових здібностей і фірмових здібностей.
7. Використовуйте кнопку палітри в заголовку аркуша, щоб вибрати кольорову схему.
8. Питайте майстра гри перед використанням tree edit mode.

## 17. Чого не змінювати під час впровадження

Не перебудовуйте акторів лише заради використання Codex II.

Не видаляйте й не створюйте предмети заново, якщо немає реальної проблеми з даними.

Не вмикайте Default Sheet Theme усім гравцям без попередження, бо це особисте налаштування і воно змінює те, як відкриваються їхні аркуші.

Не використовуйте tree edit mode під час живої гри, якщо ви справді не редагуєте макет дерева.

## 18. Відкат

Якщо хтось хоче повернутися до стандартних аркушів:

1. Для цього клієнта встановіть Default Sheet Theme назад на Default (system sheets).
2. Для будь-якого документа з явно вибраним Codex sheet відкрийте конфігурацію аркуша й виберіть стандартний аркуш.
3. Не змінюйте дані актора або предмета.

Codex II зберігає лише вибір відображення і використовує ті самі базові дані Star Wars FFG, тому відкат інтерфейсу аркушів не має видалити дані персонажа, предмети, XP або ефекти.
